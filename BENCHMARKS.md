## Memory Break-Even

LBF's logistic regression weight vector costs **64 KB fixed** (16 384 floats).
Classical BF memory scales as `N × 9.6 bits` at 1% FPR.

Break-even (model overhead = classical BF size):
- At 1% FPR target: N ≈ **54 000**
- At 1.4% FPR target (URL τ=0.7): N ≈ **59 000**

**LBF is not memory-competitive below N ≈ 54 000, regardless of dataset quality.**

---

## Key Findings

1. **Structured data is necessary but not sufficient.** AUC = 0.9991 on URLs,
   yet LBF still loses on memory for N ≤ 50k.

2. **LBF never matches classical BF's latency.** The forward pass (n-gram
   hashing + dot product) costs 370–600 ns vs 22–24 ns for classical BF.
   This is a fundamental property of the approach.

3. **FPR is determined by the model's threshold, not the backup filter.**
   The backup filter prevents false negatives; it does not reduce FPR.
   To target 1% FPR with LBF, τ must be chosen such that the model's
   own FPR at that threshold equals 1%. A convergence experiment (20→50→100
   epochs) showed that more SGD training does not close the gap to sklearn
   LBFGS: FPR plateaus around 2.5–2.7% at τ=0.7 regardless of epoch count.
   Closing the gap requires a better optimizer (LBFGS or Adam), not more
   SGD steps.

4. **Validate AUC before benchmarking.** The original words_en dataset (Step 3
   first attempt) had AUC ≈ 0.5 and produced an invalid "LBF loses" conclusion.
   The `scripts/validate_auc.py` gate prevents this error.

## Reproducing

```bash
# Dataset generation (internet required for Tranco)
python3 benchmarks/datasets/gen_uniform_int.py \
    --out benchmarks/datasets/uniform_int.tsv --n 1000000 --n-probes 200000
bash benchmarks/fixtures/gen_words_en.sh
bash benchmarks/fixtures/gen_urls_tranco.sh

# AUC gate
python3.13 scripts/validate_auc.py --tsv benchmarks/fixtures/words_en.tsv
python3.13 scripts/validate_auc.py --tsv benchmarks/fixtures/urls_tranco.tsv

# Build + run
cmake -S . -B build/release -DCMAKE_BUILD_TYPE=Release
cmake --build build/release --target bench_classical_bf bench_learned_bf
mkdir -p results
./build/release/benchmarks/bench_classical_bf \
    --benchmark_out=results/classical_bf.json \
    --benchmark_out_format=json --benchmark_repetitions=5
./build/release/benchmarks/bench_learned_bf \
    --benchmark_filter="words_en_100k|urls_tranco_100k" \
    --benchmark_out=results/learned_bf.json \
    --benchmark_out_format=json --benchmark_repetitions=5

# Plot
python3.13 scripts/plot_benchmarks.py
```
